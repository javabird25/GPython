Building¶

This article describes the process of building PyGmod from source code. You may want to do this if you’re developing on PyGmod itself. If all you want is to use or develop PyGmod addons, refer to “How to use PyGmod” guide instead.

Requirements for building¶

Windows requirements¶

  1. Visual Studio 2017 or newer with C++ workload selected

  2. CMake

  3. Python 3, preferably the latest version

  4. Git

Linux requirements¶

  1. GCC

  2. CMake

  3. Python 3

  4. Git

  5. All dependencies for building Python from source. See this AskUbuntu answer for a list of dependencies. This article in Python Developer Guide will also help.

    Note

    If you’re going to build a 32-bit version, you’ll need to get 32-bit (aka i386) version of dependencies, which is a bit tricky if you’re using a 64-bit version of your Linux distribution.

    On Debian-based distributions (Ubuntu, Linux Mint), follow this guide on Debian Wiki to install i386 packages.

    On Arch-based distributions (Manjaro), you’ll have to enable Multilib and look for lib32-* packages. You may find some of them in AUR as well.

Instructions¶

Building on Windows¶

  1. Open Developer Command Prompt for VS (most likely residing in your Start menu, folder Visual Studio).

  2. Run:

    > cd pygmod_dir\src

    substituting pygmod_dir with the full path to PyGmod source tree, for example, C:\Users\user\Downloads\PyGmod.

  3. Depending on what version of PyGmod you want to build, run:

    Release build for stable 32-bit Garry's Mod branch:
> cmake -A Win32 . && msbuild -m PyGmod.sln -p:Configuration=Release

Debug build for stable 32-bit Garry's Mod branch:
> cmake -A Win32 -DCMAKE_BUILD_TYPE=Debug . && msbuild -m PyGmod.sln -p:Configuration=Debug

Release build for x86-64 Garry's Mod branch:
> cmake -A x64 . && msbuild -m PyGmod.sln -p:Configuration=Release

Debug build for x86-64 Garry's Mod branch:
> cmake -A x64 -DCMAKE_BUILD_TYPE=Debug . && msbuild -m PyGmod.sln -p:Configuration=Debug

    Debug builds are intended for aiding to track down bugs in the C++ part of PyGmod. These builds are generally less optimized and not distributable as they’re linked against debug versions of Visual C++ runtime which ship with Visual Studio only.

  4. After a few minutes, a .pyz file should appear in src\installer-build directory. This is the PyGmod installer. You can run it to install the freshly compiled version of PyGmod.

Building on Linux¶

  1. Open a terminal in pygmod_dir/src, where pygmod_dir being the PyGmod source tree root.

  2. Depending on what version of PyGmod you want to build, run:

    Release build for stable 32-bit Garry's Mod branch:
$ cmake . && make

Debug build for stable 32-bit Garry's Mod branch:
$ cmake -DCMAKE_BUILD_TYPE=Debug . && make

Release build for x86-64 Garry's Mod branch:
$ cmake -DBITS=64 . && make

Debug build for x86-64 Garry's Mod branch:
$ cmake -DCMAKE_BUILD_TYPE=Debug -DBITS=64 . && make

  3. After a few minutes, a .pyz file should appear in src\installer-build directory. This is the PyGmod installer. You can run python3 pygmod-linuxXX.pyz to install the freshly compiled version of PyGmod.
